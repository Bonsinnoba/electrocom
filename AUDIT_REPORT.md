## Findings

- Dependencies
  - `storefront` (storefront/package.json & package-lock.json): 7 vulnerabilities (2 moderate, 5 high). Notable: `vite` (direct; versions 7.0.0–7.3.1), `rollup`, `flatted`, `minimatch`, `picomatch`, `ajv`, `brace-expansion`. Fixes available.
  - `admin-panel` (admin-panel/package.json & package-lock.json): 10 vulnerabilities (4 moderate, 6 high). Notable: `vite`, `rollup`, `lodash`, `quill`/`react-quill` (XSS), `minimatch`, `ajv`. Some fixes require major upgrades.
  - `api` (api/composer.json & composer.lock): `composer audit` returned no advisories.

- Config & repository
  - Found `.env.example` in `api`, `.env.production` files in `storefront` and `admin-panel` (potential secrets/config exposure). No `Dockerfile` detected.
  - No obvious test config files found (`phpunit.xml`, `jest`/`vitest` configs not present or not detected).

## Risk Assessment (highest priority)
- Vite vulnerabilities (high/moderate) — affects dev server and optimized deps; upgrade or apply vendor recommendations.  
- Rollup path traversal (high) — can lead to arbitrary file write if exploited during build.  
- flatted / minimatch / picomatch (high) — Denial-of-Service and prototype pollution risks.  
- react-quill / quill (XSS) — user input sanitization risk for rich text fields.  
- Presence of `.env.production` in repo — risk of secrets being committed or leaking.

## Recommended Remediation Plan (priority order)
1. Secrets first: Verify that `.env.production` files do not contain credentials. If they do, remove them from repo and rotate those secrets immediately. Add `.env*.production` to `.gitignore` if appropriate.
2. Dependency fixes (frontends): attempt safe automated fixes, then address remaining breaking changes:
   - Run `npm audit fix` in `storefront` and `admin-panel`.
   - Where fixes are major, plan an upgrade window and run full test/QA.
3. Patch or upgrade `vite` and `rollup` to versions without the advisories (follow changelogs). Consider pinning or using resolution overrides if necessary.
4. Replace or harden usage of `quill`/`react-quill` — apply sanitization (e.g., `dompurify`) on all rich text content before rendering/storing.
5. Re-run `npm audit` and `composer audit` after upgrades and confirm zero high/critical findings.
6. Run static analysis and SAST:
   - JS: `eslint .` (both `storefront` and `admin-panel`).
   - PHP: run a SAST tool (e.g., SonarCloud, Psalm, or PHPStan) to detect injection, auth, and unsafe file operations.
7. Secrets & configuration audit: scan for committed secrets (git history, uploads), check `uploads/` for executable files, and verify proper permissions on storage.
8. Add CI checks: run `npm audit --production`, `composer audit`, ESLint, and SAST in CI pipeline to prevent regressions.
